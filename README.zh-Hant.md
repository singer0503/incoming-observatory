# incoming-observatory（來襲天體觀測台）

**我們到底有多瞎？** —— 一份公開帳本，記錄人類在近地天體抵達前*實際上*提前多久看到它；
外加**第一個開源的「即時來襲預警層」**。

🌏 English → [`README.md`](README.md)

---

> 人類**有史以來**所有「在撞上地球*之前*就被發現」的小行星，預警都是以**小時**計，不是天。
> 史上最長的一次（2008 TC3）大約只有 **20 小時**，多數不到 4 小時。
> 2013 年車里雅賓斯克那顆（約 20 公尺、約 0.5 百萬噸 TNT、1,500 人受傷）給我們的預警是**零**
> —— 它從**太陽方向**飛來，地面望遠鏡在那個方向是瞎的。
>
> 這些官方機構都清楚。但**最關鍵的那個數字 ——「我們提前多久看到它？」—— 沒有任何官方表格公布。**
> 這個項目把它公開算出來，並持續更新。

## 這是什麼

三件事，全部建立在**公開**資料上（NASA/JPL CNEOS、小行星中心 MPC、ESA NEOCC）：

1. **預警時間帳本** —— 對每一顆已記錄的撞擊體、每一次近距離掠過，我們把**發現時間**
   （官方撞擊表格*不收錄*這一欄）跟**抵達時間**對接，算出並公開**預警時間**。
   招牌圖表：每一次擦肩而過，以及我們其實只提前了多麼可憐的一點點。
   關鍵是：這個數字我們**從 MPC 原始發現觀測獨立重算**（最早一筆觀測 → 撞擊），
   而不是照抄官方新聞稿 —— 而且把任何不一致**攤開來**。11 顆撞擊前被發現的小行星，
   重算結果中位數誤差約 0.5 小時；唯一對不上的那顆（2024 RW1）會被**標記，而非藏起來**。

2. **盲區稽核** —— 量化並可視化東西*為什麼*會漏掉：**太陽方向**盲區（車里雅賓斯克）、
   **長週期彗星**（預警是數月，不是數十年）、以及**星際天體**
   （1I/'Oumuamua、2I/Borisov，還有 2025 年 7 月剛來的 3I/ATLAS）。

3. **開源即時來襲預警 + 星際 triage** —— NASA 的 Scout 和 ESA 的 Meerkat 做即時撞擊預警，
   但兩者都是**閉源**的。它們吃的*輸入*（CNEOS Sentry/Scout API）卻是公開的。
   `incoming alert` 是**第一個開源、可自架**、建立在這些資料流之上的預警層 ——
   嚴重度邏輯（Palermo／Torino 等級）你看得到、跑得動、改得了。
   `incoming triage` 則依離心率標記**雙曲線／星際**天體（並算出 v∞）：
   它正確分類出全部三顆已確認的星際訪客 —— 1I/'Oumuamua、2I/Borisov、3I/ATLAS ——
   也就是「有東西剛從太陽系外面闖進來了」。

## 為什麼該有這個東西

這不是又一個軌道推算器、也不是又一個模型 —— 那些已經存在而且很優秀
（REBOUND、OrbFit、find_orb、B612 的 THOR）。經過盤點領域後確認的縫隙，是**驗證與透明這一層**：
**沒有人**公開過一份持續更新的記錄，去回答*「我們到底有沒有看見它來？看不見的那些地方又有多瞎？」*

這裡的價值**不在程式碼**（軌道數學是現成商品），而在那份**被組裝起來、附帶來源戳記的公開記錄**
—— 一個必須*對著真實世界持續維運*、而非生成一次就完事的東西。
這正是刻意設計成「AI 造不了假」的部分。

## 誠實邊界（我們絕不逾越）

這個項目的可信度*就是*產品本身，所以把界線先講清楚：

- 我們量的是「**抓到的東西，我們有多晚才抓到**」，並提供已知盲區的脈絡。
  我們**不宣稱能偵測看不見的天體** —— 你無法從「已偵測到的目錄」反推出沒偵測到的東西。
- 我們**不**做從零開始的撞擊「全程預測」（那需要我們沒有的巡天望遠鏡）。
  我們做的是預警時間統計、風險脈絡，以及對公開資料流的即時 triage。
- 我們**不**做任何「UFO／外星人」宣稱。星際天體一律以**自然天體科學**對待。
  （我們可能中性地提及 'Oumuamua 的「人造起源」假說存在且在學界有爭議 —— 那是引用，不是背書。）
- 上游 API 會變動，ESA NEOCC 的更明確標示為*實驗性*。我們會**快取並釘住**每一份快照、
  把它的雜湊記進一份 provenance 清單，所以我們公布的任何數字都可被復現。

## 快速開始

```bash
pip install -e .

# 用內建、附來源的種子資料集建立預警時間帳本（不需要連網）：
incoming ledger

# 印出招牌摘要，並輸出 outputs/warning_times.parquet 與一張圖：
incoming ledger --plot

# 從 MPC 原始觀測「獨立重算」預警時間（離線，使用快照）：
incoming recompute

# 從小行星中心即時刷新重算（會更新 data/mpc_first_obs.json）：
incoming recompute --live

# 標記雙曲線／星際天體（預設為三顆已確認的；離線使用快照）：
incoming triage
incoming triage --live "C/2025 N1"      # 從 JPL SBDB 分類任意天體

# 從 NASA 公開的 Sentry 資料流彙整出開源撞擊風險預警：
incoming alert --live
```

> **這裡的「預警時間」是什麼意思** —— 我們定義為*有史以來第一筆觀測 → 撞擊*，由最早的 MPC
> 觀測時刻算出。官方有時引用的是*發布警報 → 撞擊*（即一個天體被「認定為撞擊體」的時刻，
> 這可能晚於第一次目擊）。這個定義落差，正是重算可以合理不同的原因 —— 例如 2024 RW1，
> 從第一筆觀測算是約 10.9 小時，而官方報的約 8 小時。我們算的是定義乾淨的那個量，並把落差標出來。

每次執行都會寫出一份 `provenance.json`，記錄是哪份資料快照、哪個程式碼 commit 算出這些數字
—— 這正是本項目想要捍衛的「結果 → 來源」契約精神。

### Web 預覽（地球 · 太陽系 · 銀河系）

`incoming ledger` 同時會輸出 `web/data/ledger.json`，驅動一個 3D 檢視器（Three.js）：

```bash
cd web && python -m http.server 8000   # 然後開啟 http://localhost:8000
```

三個可切換的視角，全部由真實帳本資料驅動：**地球**視角把每一次撞擊前的偵測畫成一道
依預警時間著色的來襲軌跡（並標出太陽方向盲區的錐形）；**太陽系**視角畫出示意的
星際／雙曲線軌跡；**銀河系**視角拉遠，展示星際訪客是從哪個方向闖進來的。
它是建立在真實資料之上的展示層 —— 不會畫出任何我們沒有實測過的東西。

## 資料來源與署名

| 來源 | 用途 | 取用方式 |
| --- | --- | --- |
| **NASA/JPL CNEOS**（CAD、Sentry、Scout、SBDB） | 近距離掠過、撞擊風險、緊急目標 | 公開 JSON API，免金鑰 |
| **IAU 小行星中心（MPC）** | 發現觀測（→ 預警時間）、NEOCP 即時流 | 公開批量 + API |
| **ESA NEOCC** | 風險清單、歷史撞擊體、交叉核對 | 公開（實驗性）API |

資料歸上述機構所有，依其各自的公開使用／署名條款。本倉庫**程式碼**採 Apache-2.0。
